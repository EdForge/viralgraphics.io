Namespaces

All Visual Graphics functionality is implemented inside the VG namespace. This namespace contains a list of classes and sub-namespaces, here is a short overview of the most important namespaces and classes.

  • At the very core is the WebGL abstraction layer which consists of the following classes: ... . These classes draw all visual information inside an application to the screen.
  • VG.Core consists of core classes which are used in the UI system of Visual Graphics, like classes for Points, Rectangles and Images and so forth.
  • VG.UI contains all Widget and Layout classes used by Visual Graphics, including the Workspace class which manages the display space of an Visual Graphics application.
  • VG.Render contains all classes handling visual realtime objects, like meshes and the render pipeline.
  • VG.context is the context namespace an application is running in and the this of vgMain().

Structure of an Visual Graphics application

Every Visual Graphics application has to have a global vgMain() function which is the main entry point. A short example of an Hello World example:

function vgMain( workspace, argc, arg )
{
    var label=VG.UI.Label( "Hello World!");
    label.horizontalExpanding=true; // Center it horizontally

    workspace.content=VG.UI.Layout( label );
};

The content property of the workspace parameter has to be set to the main widget or top layout used by the application.

You can see the Hello World application in action here. For more UI examples see the tutorials.

Low-level WebGL / OpenGL Support

The VG.UI.RenderWidget class is used to visualize realtime graphics and can be embedded in any layout inside Visual Graphics (like any other UI class). VG.UI.RenderWidget can use the material system and mesh classes inside the VG.Render namespace or it can create its own low-level shaders, as demonstrated by the shadertunnel example which can be seen live here.

Examples

The examples supplied in the examples folder demonstrate various aspects of Visual Graphics:

  • The UI system and data model: AddressBook and CornellBox
  • The Render system: MTL-Demo and GameSample
  • Custom shaders: ShaderTunnel

The CornellBox example even demonstrates the integrated raytracer (WiP).
